Documentation Page

This myFlix API is one of the server-side component of the myFlix web application.

This API was built using JavaScript and Node.js. It interacts with a database that stores data about different movies, and will be accessible via the HTTP methods GET and POST.

Business Logic URL HTTP Request Method Request body data format Response body data format
Read a list of all movies /movies GET None JSON object holding data about all movies
Return data (Description, Genre, Director, Image URL, Featured: Y/N) about a single movie by Name (e.g., "The Departed") /movies/:title GET None JSON object holding data about a single movie specified by Name
Return data about a genre (description) by Name (e.g., “Thriller”) /genres/:name GET None JSON object holding data about a specified genre
Return data about a director (bio, birth year, death year) by Name (e.g., "Martin Scorsese") /directors/:name GET None JSON object holding data about a specified director
Allow new users to register /users POST JSON object holding data about user to be added (Username, Password and Email required). Username must be a minimum of five characters and be alphanumeric. Password required. Email must be valid (include a '@' and a '.') JSON object holding data about new user that was created or an error message if unsuccessful
Read a list of all users /users GET None JSON object holding data about all users
Read a user by username /users/:Username GET JSON object holding data about user you're looking for (Username) JSON object holding data about specific user you searched for
Allow users to add a movie to their FavoriteMovies list /users/:Username/movies/:MovieID POST JSON object holding data (MovieId) about movie to be added to FavoriteMovies list Show text that movie was added successfully/unsuccessfully to favorites
Allow users to update their user info (username, password, email, birthday) /users/:Username PUT JSON object holding data about user to be updated JSON object holding data about updated user and text indicating whether or not the user was successfully updated
Allow existing users to deregister /users/:Username DELETE JSON object holding data about Username of user to be removed Text indicating whether user was successfully removed or not
Allow users to remove a movie from their FavoriteMovies list /users/:Username/movies/:MovieID DELETE JSON object holding data (MovieId) about the movie(s) to be removed from FavoriteMovies list Show text that movie was added successfully/unsuccessfully from favorites
Allow existing users to login /login POST JSON object holding data about user to be logged in JSON object holding data about the user that was logged in as well as a JSON Web Token (JWT) for authorization and authentication
