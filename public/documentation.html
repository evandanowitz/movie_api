Documentation Page

This myFlix API is one of the server-side component of the myFlix web application.

This API was built using JavaScript and Node.js. It interacts with a database that stores data about different movies, and will be accessible via the HTTP methods GET and POST.

Business Logic URL HTTP Request Method Request body data format Response body data format
Return a list of all movies /movies GET None JSON object holding data about all movies
Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by Name /movies/:title GET None JSON object holding data about a single movie containing properties for a description/genre/director/image URL
Return data about a genre (description) by Name (e.g., “Thriller”) /genres/:name GET None JSON object holding data about a specific genre containing a description with data
Return data about a director (bio, birth year, death year) by name /directors/:name GET None JSON object holding data about a specific director containing a description with data
Allow new users to register /users POST JSON object holding data about user to be added JSON object holding data about new user that was created
Get list of all users /users GET None JSON object holding data about all users
Get user by username /users/:Username GET JSON object holding data about user you're looking for JSON object holding data about specific user you searched for by Username
Allow users to add a movie to their list of favorites /users/:Username/movies/:MovieID POST JSON object holding data about movie to be added to favorites Show text that movie was added successfully/unsuccessfully to favorites
Allow users to update their user info (username) /users/:Username PUT JSON object holding data about user to be updated JSON object holding data about user that was updated and text indicating whether the user was successfully updated
Allow existing users to deregister /users/:Username DELETE JSON object holding data about Username of user to be removed Text indicating whether user was successfully removed or not
Allow users to remove a movie to their list of favorites /users/:Username/movies/:MovieID DELETE JSON object holding data about movie to be removed from favorites Show text that movie was added successfully/unsuccessfully from favorites
Allow existing users to login /login POST JSON object holding data about user to be logged in JSON object holding data about the user that was logged in as well as a JSON Web Token (JWT) for authorization and authentication
